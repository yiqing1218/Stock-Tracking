# 澄鉴 A股监看

一个本地 Windows 桌面行情终端，只覆盖中国 A 股、境内 ETF 和 A 股指数。主要页面包括：

1. 自选页：按代码或名称搜索全部 A 股、ETF、指数，加入或移除自选，查看实时快照。
2. 详情页：K线与量能、指定日期分时、完整技术指标、资金/筹码/股东、公司财务、实时资讯和自定义指标。

数据通过 AkShare 和公开直连接口访问行情源。自选、公式、历史仓库、扫描任务和提醒状态都保存在本机 `data/astock_monitor.db`，不需要账号、API Key 或云端服务。

## 直接运行

本机已经创建好 `.venv` 时，双击：

```text
启动A股监看.bat
```

也可以在 PowerShell 中执行：

```powershell
.\run.ps1
```

首次在其他 Windows 电脑上运行：

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\run.ps1
```

建议使用 Python 3.12、Windows 10/11 和正常可访问公开财经网站的网络环境。

## 主导航

窗口顶部固定提供“大盘监看、条件荐股、自选股票、股票详情、量化策略/回测/指标、消息提示、数据导出”七个入口。“股票详情”只承接从自选列表打开的证券；尚未从自选进入时会保留完整分析框架，但不加载任何证券内容。

## 大盘监看

- 监看上证指数、深证成指、创业板指、沪深300、上证50、中证500、科创50和北证50，单个指数失败不会拖空整页。
- 严格使用最近当前交易日的全A快照，按昨收和主板、ST、创业板、科创板、北交所规则逐只计算上涨、下跌、平盘、涨跌停家数、成交额和中位涨幅；上市初期无涨跌幅限制证券不计入涨跌停。
- 内部分为“市场总览、行业板块、概念板块”；板块分类来源分别保存，历史从程序首次真实观测日起积累，不反向补造。
- 展示行业与概念板块完整涨跌列表；涨幅榜与跌幅榜各25只股票，右键可直接加入任意自选分组。
- 增加炸板家数、等权收益、市场波动和透明的“澄鉴市场评分”；板块页展示可解释的“澄鉴热度”，右键可同步并保存当前成分及生效区间。
- 行业/概念页可选择自选证券，以沪深300或当前板块为基准计算本地5/20/60日相对强弱；历史不足时明确显示“数据不足”。
- 全A快照支持东方财富、腾讯分批实时行情和本地短缓存回退；旧交易日数据不会混入当前交易日统计。

## 条件荐股

- 检索目标限定为全部A股，最多设置5个条件，支持且、或、非三重逻辑。
- 480项指标按“六个维度→具体指标”二级列表选择。
- 全市场扫描只读取本地历史仓库，不会在扫描时逐股联网；只计算被选中的1至5项指标，按128只一批处理并及时释放，不生成常驻的全市场480列宽表。
- 本地仓库为空时会阻止扫描并提示先同步；筛选条件可以保存为模板，结果可以导出CSV或保存为自选分组。
- 命中结果可整体保存为一个自选分组，也可右键将单只股票加入任意已有分组。

## 页面一：自选

- 搜索支持纯代码、`SH/SZ/BJ/CSI + 代码`、完整名称和部分名称，范围严格限定为 A 股、境内 ETF、A 股指数。
- 证券目录优先从本地完整缓存载入，避免上游超时后只能搜到少数内置证券。
- 支持自选分组；可选择加入目标分组，并通过“分组管理”新建、重命名或删除分组。
- 双击行进入详情；右键菜单提供打开详情、上移、下移、移到分组和移除。
- 展示最新价、涨跌幅、成交额、换手率、量比、PE、PB、总市值等公开字段。
- 最新价、涨跌幅、涨跌额、成交额、换手率、量比、PE、PB、总市值和评分等数字列均可点击表头排序；缺失值固定排在末尾。
- 新增“评分”列，与详情页经典系统指标的加权六维评分保持同一口径。
- 表格直接显示本次行情实际采用的数据源，不再显示“详情/移除”按钮。
- 行情每 60 秒后台刷新，网络请求不阻塞界面；首次完整证券列表同步后使用 24 小时缓存。

## 页面二：详情

### 行情走势

- 红涨绿跌的日K、成交量、MA5/MA20、布林带和 MACD。
- 分时和K线位于同一“行情”页；支持日K、周K、年K切换。鼠标滚轮或图内＋/－按钮缩放，←/→按钮和键盘左右方向键平移可见区间。
- K线可从20个经典价格叠加项中多选；副图可切换MACD、RSI、KDJ、CCI、OBV、MFI和成交量。
- 日线自动合并北京时间当日实时报价；右侧可切换前复权、不复权和后复权。
- K线标注股票/ETF可取得的除权、除息与分红日；ETF复权增加腾讯数据源，并可由新浪原始日线与累计分红序列在本地重建前/后复权。
- 双击任意K线会直接进入对应交易日的1分钟分时；行情走势页不再显示筹码分布图。
- 趋势/震荡状态、综合多维评分、历史波动率、回撤与六维状态摘要；六个维度会完整显示，每个维度都汇总该维度下所有可用指标，而不是挑选单一指标。MACD、RSI、ADX、均线、BOLL、ATR、OBV、MFI、CMF 和回撤等经典指标采用更高预设权重，扩展输出采用较低权重，避免大量相似输出淹没经典信号。

### 历史分时

- 可用日历选择某一天，并切换 1、5、15、30、60 分钟周期，默认1分钟；选择证券、日期或周期后自动加载，不再保留“加载分时”按钮。
- 北京时间当日采用实时分时，20秒短缓存后重新请求；历史日期使用长期本地缓存。
- 分时图显示日内 K 线、成交量和时间轴；以昨收为 0.00% 基准绘制零线，右侧同时显示价格和涨跌百分比，悬浮信息也显示涨跌幅。非交易日或接口范围外会明确提示。
- 公开接口的 1 分钟数据通常只保留最近 5 个交易日，较长周期的可用日期一般更长。

### 全部指标

内置手工指标目录之外，程序还会加载 Pandas TA Classic 的公开指标及多输出结果；历史数据充足时通常可展开为 400 多个系统指标条目，实际数量会随数据长度和库版本变化。所有条目统一归入趋势、动量、波动、量能、情绪、风险六个维度：

- 趋势：MA/EMA/WMA、MACD、DMI/ADX、Aroon、SAR、Supertrend、一目均衡、BBI、DMA、线性斜率。
- 动量：RSI、KDJ、Stochastic、Williams %R、ROC、MOM、CCI、CMO、TRIX、PPO、Ultimate Oscillator。
- 波动：BOLL、ATR/NATR、历史波动率、Keltner、Donchian、Chaikin Volatility、Ulcer Index。
- 量能：OBV、MFI、CMF、A/D、Chaikin Oscillator、PVT、Force Index、EMV、量比、VWAP。
- A股常见情绪：BIAS、PSY、AR、BR、CR。
- 风险：多周期收益、滚动夏普、历史 VaR、当前回撤、偏度、峰度。
- 指标页分为“K线指标、K线形态、财务指标”。形态页内置50种常见单K、双K、三K和多K结构，命中项高亮、未命中项变暗，并附完整解释。
- 财务指标页展示经典利润质量、现金流、资产质量、负债与费用指标，并计算可核验的同比、环比和自身历史Z分数；缺失值不会推算。
- 指标名称和说明单元格都带完整悬浮提示，鼠标停留后可阅读全文。
- 每个指标右侧都有收藏星标；点亮右上角星标后只显示已收藏指标，收藏状态保存在本地数据库。
- 指标表新增“来源”列，直接标识“系统”或“自定义”。
- 每个指标的说明包含计算口径、常见解读、维度限制、来源和六维评分预设权重，不再只显示一句简称。
- 为降低详情首屏延迟，行情和经典指标优先显示；数百个扩展指标只在首次进入“全部指标”时单进程计算，并保存 10 分钟轻量快照缓存。扩展计算结束后释放中间宽表，不长期占用数百列内存。

### 简况（F10）、资金、筹码、盘口、财务

原合并页已经拆分。详情页结构为“行情、指标、简况（F10）、资金、筹码、盘口、财务、企业公告/新闻/资讯、公司事件、行情提醒设置”。

- 主力资金流：优先读取东方财富个股资金流，再尝试东方财富全市场排行和同花顺资金流；公开源不可用时由本地 OHLCV 收盘位置模型估算，并明确标注“非逐笔主力”。
- 筹码分布：只展示东方财富公开筹码接口的原始结果。接口无返回时，获利比例、平均成本、成本区间与集中度均显示“暂无数据”，不使用成本模型补造。
- 盘口：展示腾讯公开五档买卖盘快照，明确标注“非Level-2”；不构造逐笔、队列、撤单或封单强度。
- 主要股东：最新可用定期报告的股东名称、数量、比例、性质和披露日期。

“主力资金流”是成交单大小的统计口径，不等于机构真实持仓；“主要股东”是定期披露口径，也不代表实时仓位。应用在页面中明确分开显示。

- 企业概况：上市时间、行业、总股本、流通股、公司名称、注册地址等可取得字段。
- 主营业务：主营范围、产品与经营介绍等公开资料。
- 财务：展示摘要图表、资产负债表、利润表和现金流量表，以及质量指标、同比、环比和标准化观察；接口不提供可核验版本链时不猜测修订内容。
- F10深度：基于已经加载的披露数据归纳盈利、成长、偿债、现金流质量及简单风险线索，不重复下载整套财务宽表。
- ETF 和指数没有单一上市公司的财务报表，页面会显示“不适用”。

### 实时资讯

- 自动汇总东方财富个股新闻与 Bing 联网新闻搜索结果，转换成“时间、来源、标题、摘要”条目。
- 自动去重并保留 10 分钟缓存；双击条目可打开原文。

## 自定义指标

公式解析器使用 AST 白名单，不调用 Python `eval`，不能访问文件、网络、模块或对象属性。

基础变量：

```text
open, high, low, close, volume, amount, turnover,
pct_change, returns, typical, hl2, ohlc4
```

可用函数：

```text
SMA(x,n) EMA(x,n) WMA(x,n) SUM(x,n) STD(x,n)
HHV(x,n) LLV(x,n) REF(x,n) DIFF(x,n) ROC(x,n)
ZSCORE(x,n) TS_RANK(x,n) CORR(x,y,n)
IF(condition,a,b) CROSS(a,b) MAX(a,b) MIN(a,b)
ABS(x) LOG(x) SQRT(x) CLIP(x,min,max)
```

示例：

```text
EMA(close, 12) - EMA(close, 26)
(close / SMA(close, 20) - 1) * 100
ZSCORE(returns, 20) + ZSCORE(volume, 20)
IF(CROSS(SMA(close, 5), SMA(close, 20)), 1, 0)
```

公式校验、保存和绘图位于主导航“量化策略/回测/指标”的“指标”子页。保存时可选择六个维度之一，并选择是否加入“全部指标”库；加入后会显示为“自定义”来源并参与对应维度的综合状态计算。

## 行情提醒与消息中心

- 详情页“行情提醒设置”支持价格、涨跌幅、成交额、换手率、量比、放量、近期高低点、涨跌停状态和短时间急涨急跌规则。
- 行情规则仅在北京时间A股交易时段每30秒批量检查。统一提醒表持久化上次值、真假状态、冷却时间和事件快照，重启后不会把仍然成立的条件误判为新触发。
- Windows系统托盘保持提醒服务运行；关闭主窗口后可从托盘恢复或退出，触发时发送桌面通知。
- 消息中心自动归类自选股近3日资讯、公告、业绩预告、财报、分红、解禁、减持、质押、回购、停复牌、龙虎榜、大宗交易和融资余额相关条目，并提供未读数与已读管理。
- 双击带有效 `http/https` 来源的消息会标记已读并直接打开原始网页。

## 公告与公司事件时间线（阶段四）

- 详情页“公司事件”按公告日、事件/生效日、类型、官方来源、重要度和关键字筛选，支持已读状态、研究备注、原始载荷和原始网页。
- 公告标题使用稳定规则分类，不调用生成式模型；情绪默认“未知”，重要度显示透明规则评分，不把标题强行判成利好或利空。
- 巨潮公告、分红、限售解禁、质押和股东户数等适配器互相隔离并支持增量游标、重试与取消；单一来源失败不会清空其他来源结果。
- 官方来源优先，原始载荷和来源链保留；回购方案、首次实施、进展、完成、取消等阶段不会错误合并。
- 首次同步只建立基线不推送旧事件；后续新增事件进入消息中心，并可在K线按真实公告/生效日期显示聚合标记。

## 量化策略/回测/指标（阶段六）

- “量化”子页预留；“回测”只读取本地历史仓库，不会在运行过程中联网补数。支持 `ENTRY` / `EXIT` / `SCORE`、日期范围、单股/自选组/动态选股组/本地全部A股/本地全部ETF、固定持有、止盈止损、等权/固定/评分权重、最大持仓和日/周/月调仓。
- 默认信号在T日收盘后确认、T+1开盘执行，也提供T+1收盘和OHLC4近似VWAP敏感性模式并明确近似口径。
- 撮合处理A股T+1、整手、停牌/无量、一字涨跌停、ST特殊涨跌幅、最少上市日、佣金最低收费、股票卖出印花税、ETF免印花税和滑点；未成交订单逐笔保留原因。
- 输出累计/年化/基准/超额收益、波动率、最大回撤及持续期、Sharpe、Sortino、Calmar、Beta、Alpha、信息比率、胜率、盈亏比、平均盈亏、持仓时间、连续亏损、换手、成本、滑点、平均仓位、年度/月度收益、交易、持仓、资金曲线和信号原始数据。
- 策略定义、运行配置哈希、数据版本、成交/未成交、每日权益/持仓和指标均持久化；可取消，报告明确全市场当前证券清单可能带来的幸存者偏差。内置均线、突破和均值回归三个教学模板，不连接真实交易。

## 本地历史仓库与数据导出

- `securities` 与 `daily_bars` 按证券、交易日、复权方式唯一保存，使用 WAL、批量 UPSERT 和短连接；网络失败不会删除旧数据。
- 支持当前自选、全部A股、全部ETF、主要指数和全部证券的增量、完整、修复同步；任务状态、失败明细、取消标记和质量问题均持久化。
- “数据导出”页提供仓库概况、旧CSV缓存导入、后台同步、取消、质量检查和按范围/日期/复权导出CSV。导出按证券流式写入，不建立全市场内存宽表。

```powershell
.\.venv\Scripts\python.exe -m astock_monitor.sync_cli status
.\.venv\Scripts\python.exe -m astock_monitor.sync_cli daily --scope stocks
.\.venv\Scripts\python.exe -m astock_monitor.sync_cli full --scope etfs
.\.venv\Scripts\python.exe -m astock_monitor.sync_cli validate
```

## 数据与计算口径

- 详情页把“行情首屏、资金与公司资料、联网资讯、扩展指标”拆成独立后台任务；页面不会等待所有慢接口结束后才显示 K 线。
- Pandas TA Classic 在 Windows 上强制单进程运行，避免按 CPU 数量复制 Python/Pandas 进程造成高内存和递归启动风险。
- 实时行情按“腾讯行情直连 → 东方财富直连 → 新浪行情直连 → 本地历史缓存”自动回退，优先使用当前网络下延迟更低的来源。
- 大盘监看优先用东方财富全A/行业板块单页直连接口，失败后回退 AkShare 封装和本地短缓存；避免为一次刷新逐页下载整个市场。
- 日线按品种使用下列 AkShare 接口自动回退：
  - 股票：`stock_zh_a_hist` → `stock_zh_a_hist_tx` → `stock_zh_a_daily`
  - ETF：`fund_etf_hist_em` → `stock_zh_a_hist_tx` → 原始日线+累计分红本地复权；不复权再回退 `fund_etf_hist_sina`
  - 指数：`stock_zh_index_daily_em` → `stock_zh_index_daily_tx` → `stock_zh_index_daily` → `index_zh_a_hist`
- 分时接口：股票使用 `stock_zh_a_hist_min_em` / `stock_zh_a_minute`；ETF 使用 `fund_etf_hist_min_em` / `stock_zh_a_minute`；指数使用 `index_zh_a_hist_min_em` / `stock_zh_a_minute`。
- 证券目录：`stock_info_a_code_name`、`fund_etf_spot_em`、`stock_zh_index_spot_em`。
- 公司和财务：`stock_ipo_info`、`stock_individual_info_em`、`stock_profile_cninfo`、`stock_individual_basic_info_xq`、`stock_zyjs_ths`、`stock_financial_abstract_ths`、`stock_financial_analysis_indicator`。
- 资金和持仓：`stock_individual_fund_flow`、`stock_individual_fund_flow_rank`、`stock_fund_flow_individual`、明确标注的本地 OHLCV 资金估算、`stock_cyq_em`、`stock_main_stock_holder`。筹码不做本地估算。
- 资讯：`stock_news_em` 加 Bing 新闻 RSS 搜索；不需要 API Key。
- AkShare 接口定义与字段以 [AkShare 股票数据文档](https://akshare.akfamily.xyz/data/stock/stock.html) 和 [ETF 分时文档](https://akshare.akfamily.xyz/data/fund/fund_public.html) 为准。
- 指标分类参考 [TA-Lib 官方函数目录](https://ta-lib.org/functions/)；扩展指标通过 [Pandas TA Classic](https://xgboosted.github.io/pandas-ta-classic/indicators.html) 计算，项目原有指标继续使用 pandas/numpy 实现。
- 滚动均值、标准差和指数加权使用 [pandas Window API](https://pandas.pydata.org/pandas-docs/stable/reference/window.html)。
- 日线默认前复权；指标基于 OHLCV、成交额、换手率等公开字段。
- 所有“今天”、交易状态、缓存年龄和界面时间均统一按 `Asia/Shanghai` 北京时间计算，不受 Windows 当前时区影响。
- 指数和 ETF 不显示个股股东/筹码披露，避免制造不存在的数据。
- 网络失败时优先读取最近一次成功缓存，并在界面显示提示。
